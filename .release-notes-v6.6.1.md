# v6.6.1 - Three-phase dashboard and setpoint fixes

Bugfix release focused on three-phase usability while preserving existing single-phase behaviour.

## Fixed

- Reworked the managed and bundled three-phase dashboard to use native Lovelace entity rows and markdown tables instead of `custom:template-entity-row` and `card_mod`.
- Kept the existing aggregate `sensor.electricity_planner_grid_setpoint` unchanged and added three-phase-only setpoint sensors:
  - `sensor.electricity_planner_grid_setpoint_phase_1`
  - `sensor.electricity_planner_grid_setpoint_phase_2`
  - `sensor.electricity_planner_grid_setpoint_phase_3`
- Added `phase_grid_setpoints` to `sensor.electricity_planner_decision_diagnostics` attributes so automations can read the phase decisions from the diagnostics payload too.
- Added optional per-phase grid power sensors in the three-phase setup flow and dashboard.
- Removed the invalid configuration consistency rule that blocked `max_car_power` from being greater than `max_grid_power`; EV charger capacity can exceed the grid import cap when solar or battery power contributes.

## Compatibility

- No config schema migration required.
- Single-phase setup and the aggregate grid setpoint sensor are unchanged.
- Existing automations using `sensor.electricity_planner_grid_setpoint` continue to work.
- Managed dashboards re-save on integration reload via dashboard managed version `36`.

## Validation

- Docker test suite: `516 passed`.
