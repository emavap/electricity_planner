# v6.6.2 - Sunny-day battery charge hysteresis

Bugfix release for battery grid-charging stability.

## Fixed

- Added a 2% SOC hysteresis band to the sunny-day grid-charging ceiling while battery grid charging is already active.
- Prevents rapid ON/OFF flapping around the reduced sunny-day max SOC threshold, for example around 35% when high solar is forecast.
- Keeps the configured sunny-day max SOC as the restart gate: once charging stops, it will not restart until SOC falls back below the configured threshold.
- Leaves the normal max SOC ceiling, price threshold logic, manual overrides, and entity IDs unchanged.

## Compatibility

- No config schema migration required.
- No dashboard managed-version bump required.
- Existing automations and dashboards continue to work unchanged.

## Validation

- Added regression tests covering active sunny-day hysteresis, stop-above-band behaviour, and no restart at the boundary while charging is already off.
- Docker test suite: `519 passed`.
